# Поліглот для NVDA

Поліглот — це глобальний додаток для NVDA, призначений для швидкого та гнучкого багатомовного перекладу. Він може перекладати вибраний текст, вміст буфера обміну та останній промовлений текст NVDA, а також перехоплювати промовлений вміст для автоматичного перекладу в режимі реального часу.

Цей додаток побудовано на основі архітектури динамічного сервісу перекладу. Відповідні сервіси оголошують про свої можливості та схему конфігурації, а інтерфейс налаштувань генерується на основі цієї схеми під час виконання. Це дозволяє зберегти компактність основного додатка та водночас спрощує додавання нових сервісів перекладу.

## Що він вміє

- Перекладає вибраний текст, текст буфера обміну та останню промовлену фразу NVDA.
- Надає командний шар. `NVDA+Alt+Z`. Натисніть `H` у командному шарі, щоб показати довідку для команд.
- Підтримує автоматичний переклад останнього промовленого тексту NVDA в реальному часі.
- Містить інтелектуальний фільтр мовлення, що дозволяє уникнути перекладу ролей, станів та форматування.
- Зберігає кеш перекладу, щоб зменшити кількість повторних запитів.
- Може автоматично копіювати результати перекладу в буфер обміну.
- Дозволяє перемикатися між сервісами перекладу та мовами, не відриваючись від клавіатури.
- Відкриває спеціальний інтерактивний діалог для тривалих або багаторазових завдань з перекладу.

## Встановлення

Рекомендований шлях встановлення — магазин додатків NVDA. Ви також можете встановити додаток вручну:

1. Завантажте найновіший пакет `.nvda-addon` зі [сторінки випусків](https://github.com/cary-rowen/polyglot/releases).
2. Відкрийте завантажений файл.
3. Підтвердьте встановлення в NVDA.
4. Перезапустіть NVDA, коли з’явиться відповідне повідомлення.

## Швидкий старт

1. Відкрийте `меню NVDA -> Параметри -> Налаштування -> Поліглот`.
2. Виберіть сервіс перекладу та переконайтеся, що він увімкнений.
3. Налаштуйте всі необхідні облікові дані для цього сервісу.
4. Виберіть мову оригіналу та мову перекладу.
5. За бажанням увімкніть копіювання в буфер обміну та інтелектуальний фільтр мовлення.
6. Натисніть `NVDA+Alt+Z`, а потім скористайтеся однією з наведених нижче клавіш командного шару. Натисніть `H` у цьому шарі, щоб відкрити довідку щодо команд.

## Командний шар

Натисніть `NVDA+Alt+Z`, щоб увійти в командний шар. Короткий звуковий сигнал підтверджує, що командний шар активний. Натисніть `H`, щоб показати довідку щодо команд у шарі. Більшість команд виконуються одноразово, після чого командний шар закривається. Команди перемикання мови та сервісу перекладу залишаються в цьому шарі, тож ви можете продовжувати циклічно перемикатися між ними. Перемикання сервісу перекладу здійснюється лише серед увімкнених сервісів перекладу.

| Комбінація | Дія |
| --- | --- |
| `T` | Перекласти поточний вибраний фрагмент. |
| `Shift+T` | Перекласти поточний вибраний фрагмент у зворотному порядку. |
| `B` | Перекласти текст із буфера обміну. |
| `Shift+B` | Перекласти текст із буфера обміну у зворотному порядку. |
| `L` | Перекласти останній промовлений текст NVDA. |
| `Shift+L` | Перекласти останній промовлений текст NVDA, у зворотному порядку. |
| `S` | Наступна мова оригіналу. |
| `Shift+S` | Попередня мова оригіналу. |
| `G` | Наступна мова перекладу. |
| `Shift+G` | Попередня мова перекладу. |
| `E` | Наступний увімкнений сервіс перекладу. |
| `Shift+E` | Попередній увімкнений сервіс перекладу. |
| `W` | Поміняти місцями мови оригіналу та перекладу. |
| `A` | Оголосити поточний сервіс перекладу та мовну пару. |
| `C` | Скопіювати останній результат перекладу. |
| `V` | Увімкнути/вимкнути автоматичний переклад. |
| `I` | Відкрити діалог інтерактивного перекладу. |
| `O` | Відкрити налаштування додатка. |
| `X` | Очистити кеш перекладу. |
| `H` | Показати довідку щодо команд. |

## Інтерактивний діалог перекладу

Інтерактивний діалог призначений для перекладу довших текстів та роботи, що передбачає кілька етапів перекладу.

- Відкрийте його з командного шару за допомогою клавіші `I`.
- Виберіть активний сервіс перекладу, мову оригіналу та мову перекладу, не виходячи з діалогу.
- Неактивні сервіси перекладу можна налаштувати в налаштуваннях додатка, але вони не показуються в цьому діалозі.
- Для сервісів типу LLM налаштуйте модель та шаблон запиту безпосередньо в діалозі.
- Натисніть `Ctrl+Enter` у полі введення тексту для перекладу, щоб виконати переклад.
- Скопіюйте результат або очистіть обидва поля, не відкриваючи діалог заново.

## Посібник з налаштувань

### Загальні налаштування

- `Копіювати результат перекладу в буфер обміну`: копіює результат перекладу після успішного запиту.
- `Увімкнути розумний фільтр мовлення`: під час перекладу промовленого тексту NVDA пропускає неінформативні елементи, такі як ролі, стани, орієнтири та деталі форматування, де це можливо.
- `Очистити кеш`: очищає постійний кеш перекладів і показує поточну кількість елементів у підписі кнопки.

### Загальні налаштування сервісу перекладу

Більшість сервісів мають однаковий набір параметрів:

- `Увімкнути цей сервіс перекладу`: визначає, чи доступний сервіс для запитів на переклад, перемикання сервісів у командному шарі та в інтерактивному діалозі. Вимкнені сервіси залишаються видимими та доступними для налаштування в параметрах.
- `Мова оригіналу` та `Мова перекладу`
- `Режим проксі`: використовувати системні налаштування проксі або вимкнути використання проксі
- `Тайм-аут запиту`

Якщо сервіс перекладу повідомляє про виявлену мову оригіналу, додаток також надає:

- `Автозаміна, якщо мова оригіналу збігається з мовою перекладу`: корисно, коли мова оригіналу налаштована на автоматичне визначення
- `Мова для зміни`: додаткова мова, яка використовується під час автоматичного перемикання

### Особливості роботи функції автоматичного перекладу

- Автоматичний переклад діє на промовлений текст NVDA, отриманий через синтезатор мовлення.
- Додаток блокує власні промовлені повідомлення, щоб уникнути циклів перекладу.
- Якщо автоматичний переклад не вдається виконати тричі поспіль, він вимикається автоматично.
- Розумний фільтр мовлення впливає переважно на переклад промовленого тексту, а не на стандартний ручний переклад тексту.

### Параметри LLM та специфічні налаштування для багатомовних моделей

Деякі сервіси перекладу мають додаткові елементи керування:

- `Ollama 1` та `Ollama 2` надають два окремі збережені профілі для різних локальних або віддалених конфігурацій Ollama.
- `OpenRouter` надає URL API, ключ API, попередньо встановлену модель, власну назву моделі, шаблон підказки та власні підказки.
- Сервіси `Ollama` надають URL API, назву моделі, опціональний ключ API, шаблон запиту та власні запити.
- `Google Translate (Поліглот)` надає настроювану URL кінцевої точки та поле для ключа API.
- `Google Translate (без ключа)` пропонує опціональну можливість увімкнення/вимкнення дзеркального сервера.

## Офлайн-переклад Chrome AI

Поліглот може використовувати вбудований API перекладача Chrome для офлайн-перекладу. Переклад виконується ізольованим локальним екземпляром Chrome, тому текст не надсилається сторонній службі перекладу.

### Вимоги

- Необхідно встановити Google Chrome.
- Рекомендується використовувати Chrome версії 138 або новішої.
- Для першого використання перекладу потрібно підготувати локальну модель перекладу. Додаток може завантажити модель через свій менеджер моделей, або ви можете дозволити Chrome завантажити її самостійно.

### Як використовувати

У налаштуваннях додатка виберіть «Chrome AI (Офлайн)», а потім вкажіть мову оригіналу та мову перекладу. Для роботи Chrome AI необхідно чітко вказати мову оригіналу; функція «Автовизначення» для цього сервісу недоступна, тому поліглот може перевірити наявність необхідної моделі перед запуском Chrome.

Під час першого використання, якщо необхідна модель не встановлена, Поліглот запитує, як діяти. Виберіть «Так», щоб завантажити та встановити модель за допомогою менеджера моделей Поліглот; використовуйте цей варіант, якщо сервіс завантаження моделей Chrome працює повільно, заблокований або нестабільний у вашій мережі. Виберіть «Ні», щоб дозволити завантаження моделі через Chrome. Виберіть «Скасувати», щоб скасувати поточний переклад. Після підготовки моделі переклад продовжиться автоматично.

### Мережа та моделі

Переклад виконується локально. Моделі можна встановити за допомогою менеджера моделей Поліглот або завантажити через Chrome. Якщо служба завантаження моделей у Chrome працює повільно або недоступна у вашій мережі, виберіть «Так» у спливаючому діалозі або відкрийте менеджер моделей Поліглот ChromeAI з меню «Інструменти» NVDA, щоб заздалегідь встановити або видалити офлайн-моделі.

### Конфіденційність і дані

Поліглот використовує окремий каталог даних Chrome для Chrome AI, тому це не впливає на ваш звичайний профіль Chrome. Моделі, дані кешу та дані виконання зберігаються, щоб уникнути повторного завантаження.

Початкове розташування:

```текст
%LOCALAPPDATA%\Polyglot\ChromeAI
```

Якщо змінна середовища `LOCALAPPDATA` відсутня, Поліглот використовує каталог `polyglot_chrome_ai`, розташований у каталозі конфігурації NVDA.

Коли NVDA завершує роботу, Поліглот закриває запущений екземпляр Chrome.

### Обмеження

- Перелік підтримуваних мов та мовних пар визначається API «перекладачем» браузера Chrome.
- Для роботи Chrome AI необхідно вказати мову оригіналу; для цього сервісу функція «Автовизначення» недоступна.
- Перед першим використанням модель потрібно підготувати; швидкість завантаження моделі може залежати від стану мережевого з’єднання.
- Якщо API «Перекладач» недоступний, оновіть Chrome або переконайтеся, що відповідна функція Chrome увімкнена.

## Огляд сервісу перекладу

Наразі додаток містить такі сервіси:

| Сервіс | Облікові дані | Примітки |
| --- | --- | --- |
| `Baidu Translate` | Ідентифікатор та секретний ключ додатка Baidu | Стандартна інтеграція API постачальника. |
| `Caiyun` | Токен Caiyun | Стандартна інтеграція API постачальника. |
| `Chrome AI (офлайн)` | Немає | Використовує вбудований API перекладача Chrome з локальними моделями; мову оригіналу потрібно вказати вручну. |
| `DeepL` | Ключ API DeepL | Стандартна інтеграція API постачальника. |
| `Google Translate (без ключа)` | Немає | Підтримує опціональне вмикання/вимикання дзеркальної кінцевої точки. |
| `Google Translate (Поліглот)` | Настроюваний ключ API та кінцева точка | Постачається з початковими значеннями кінцевої точки в коді; доступність залежить від стану сервісу. |
| `Lingva Translate` | Немає | Публічна кінцева точка Lingva, у відповідях відсутні дані про виявлення мови. |
| `Microsoft Translator (без ключа)` | Немає | Автоматично отримує тимчасовий токен. |
| `Niutrans` | Ключ API Niutrans | Стандартна інтеграція API постачальника. |
| `Ollama 1` | URL Ollama, назва моделі, опціональний ключ | Перший збережений профіль Ollama. |
| `Ollama 2` | URL Ollama, назва моделі, опціональний ключ | Другий збережений профіль Ollama. |
| `OpenRouter` | Ключ API OpenRouter | Підтримує попередні налаштування моделей та редаговані шаблони підказок. |
| `Tencent Translate` | Секретний ідентифікатор та секретний ключ Tencent | Стандартна інтеграція API постачальника. |
| `Tencent Translate (Поліглот)` | Ім’я користувача та пароль NVDACN | Маршрут Tencent на базі Поліглот. |
| `VIVO Translate` | Ім’я користувача та пароль NVDACN | Обмежений набір мов, відсутнє автоматичне визначення мови оригіналу. |
| `Volcengine (Поліглот)` | Ім’я користувача та пароль NVDACN | Маршрут Volcengine на базі Поліглот. |
| `Yandex Translate` | Немає | Пункт кінцевого доступу відкритого типу, відсутність звітів про визначену мову. |

## Сприяння

Вітаються внески щодо коду, документації, локалізації, тестування та інтеграції сервісів перекладу.

- Проблеми: [GitHub Issues](https://github.com/cary-rowen/polyglot/issues)
- Випуски: [GitHub Releases](https://github.com/cary-rowen/polyglot/releases)

При додаванні нового сервісу перекладу:

1. Створіть модуль у каталозі `addon/globalPlugins/polyglot/services/engines/`.
2. Реалізуйте інтерфейс `TranslationEngine` або, для HTTP-engines, розширте клас `BaseHttpEngine`.
3. Поверніть специфікацію конфігурації з `getConfigSpec()`, якщо сервісу потрібні налаштування.
4. Використовуйте типи елементів управління, що підтримуються, з файлу `views/factory.py`: `choice`, `text`, `password`, `checkbox` та `spinctrl`.
5. Переконайтеся, що сервіс правильно показується на панелі динамічних налаштувань, а також, якщо він увімкнений, у перемиканні командного шару та інтерактивному діалозі.
